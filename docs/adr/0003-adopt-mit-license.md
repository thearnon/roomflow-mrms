# Adopt MIT License

RoomFlow will use the MIT License because the project is intended to be public,
portfolio-friendly, and reusable by people who want to learn from it or build on
it. This keeps the legal posture simple and permissive while still requiring
the license notice to be preserved.

## Considered Options

- **No license**: keeps all rights reserved by default, but conflicts with the
  goal of letting others reuse or adapt the project.
- **MIT License**: simple, widely understood, permissive, and suitable for
  public learning and portfolio projects.

## Consequences

Future public copies can use, modify, distribute, and build on RoomFlow under
the MIT terms. This decision should be revisited only if the project later needs
a different licensing model for a specific distribution reason.
